# 프로젝트 계획 (plan.md)

## 개요
기존 "우리반 교실" 자리 배치/점수 관리 웹앱(`장등초 자리.html`)을 기반으로,
1) 다른 페이지로 이동하는 기능과
2) 학생이 모은 하트(점수)로 자신의 아바타를 꾸미는 상점 기능
을 추가한다.

원본 파일은 Desktop에 그대로 두고, `클로드` 프로젝트 폴더 안에서 별도로 확장 작업을 진행한다.

## 페이지 구성
1. **index.html** (기존 "장등초 자리.html"을 복사해서 시작)
   - 우리반 교실 자리 배치 / 모둠 점수 관리 화면 (기존 기능 그대로 유지)
   - 각 학생 책상 카드에 "내 아바타 보기" 아이콘 버튼 추가
   - 클릭 시 `avatar.html?student=학생번호` 로 이동

2. **avatar.html** (신규)
   - 학생 개인 아바타 꾸미기 + 상점 페이지
   - 상단: 뒤로가기 버튼, 학생 이름, 보유 하트 수
   - 중앙: 현재 아바타 미리보기 (장착한 아이템 반영)
   - 하단/사이드: 탭 전환
     - **상점 탭**: 카테고리별(모자/상의/하의/액세서리/무기) 아이템 목록, 하트로 구매
     - **보관함 탭**: 이미 구매한 아이템 목록에서 장착/해제

## 기능 목록

### 기존 기능 (유지)
- 21명 학생 5모둠 자동 배치
- 모둠 카드 드래그 이동, 학생 자리 교환(드래그&드롭)
- 학생별 하트 점수 +/- 버튼
- 모둠 점수 합산 + 슬라임 캐릭터 반응(기쁨/슬픔)
- 랜덤 자리바꾸기, 점수 초기화
- 학생 이름 설정(모달)
- localStorage 저장

### 신규 기능
- [ ] 자리판 → 아바타 페이지 이동 (학생 책상의 아바타 아이콘 클릭)
- [ ] 아바타 페이지 → 자리판으로 돌아가기 버튼
- [ ] 학생별 아바타 상태 저장: 보유 아이템 목록(inventory), 현재 장착 아이템(equipped: 모자/상의/하의/액세서리/무기)
- [ ] 시작 아바타: 민소매 + 반바지 기본 착장 (무료 기본 아이템, 처음부터 장착됨)
- [ ] 상점: 카테고리별 아이템 목록 표시, 하트 가격 표시
- [ ] 아이템 구매: 보유 하트 확인 → 차감 → 보관함에 추가
- [ ] 보관함: 구매한 아이템 장착/해제, 현재 장착 중인 아이템 표시
- [ ] 아바타 미리보기: 장착 아이템에 따라 실시간으로 캐릭터 모습 변경
- [ ] 하트 부족 시 구매 불가 안내 (토스트 메시지)
- [ ] 학생 개인 기기에서 avatar.html 접속 → 이름 선택으로 본인 확인 (최초 1회, 이후 자동 기억)
- [ ] Firebase Firestore 연동: 교사 화면 ↔ 학생 화면 실시간 동기화
- [ ] Firebase Hosting으로 배포하여 공개 URL 생성

## 백엔드 & 배포 (Firebase)
학생들이 각자 폰·태블릿으로 접속해서 실시간으로 하트 확인/아바타 구매를 하려면,
데이터를 브라우저에만 저장하는 localStorage로는 안 되고 **모두가 공유하는 클라우드 데이터베이스**가 필요하다.
→ **Firebase**(구글 무료 클라우드 서비스)로 구축한다.

- **Firestore**: 실시간 클라우드 데이터베이스. 교사가 자리판에서 점수를 올리면, 학생 화면에도 실시간으로 반영됨.
- **Firebase Hosting**: 완성된 웹앱을 인터넷에 무료로 배포 (예: `https://우리반.web.app` 같은 공개 주소 생성). 학생들은 이 주소로 각자 기기에서 접속.
- **학생 식별 방법 (로그인 없이, 간단하게)**: 학생이 처음 avatar.html에 접속하면 "나는 누구인가요?" 화면에서 자기 이름을 목록에서 선택 → 이후엔 그 기기(브라우저)에 자동 기억되어 매번 다시 고를 필요 없음. 비밀번호 없이 이름만 선택하는 방식이라 초등학생도 쉽게 사용 가능 (단, 다른 학생 이름을 실수/장난으로 고를 수 있는 점은 감안).
- **준비물 (사용자가 직접 해야 하는 부분)**:
  1. 구글 계정으로 Firebase 콘솔(firebase.google.com/console)에서 무료 프로젝트 생성
  2. 생성된 프로젝트의 설정값(API key 등)을 코드에 붙여넣기 — 복사/붙여넣기만 하면 됨, 코딩 지식 불필요
  3. 배포는 구글 로그인 절차가 필요해서 제가 대신 할 수 없는 단계 — 제가 명령어를 하나씩 알려드리면 그대로 따라 실행하시면 됨

## 데이터 구조 (Firestore)
기존 localStorage의 state 객체를 그대로 Firestore의 문서(document) 하나로 옮겨서 사용 (구조는 거의 동일, 저장 위치만 클라우드로 이동):
```
Firestore 문서: classroom/state
{
  students: [...],
  groups: [...],
  scores: { ... },
  avatars: {
    [studentIdx]: {
      inventory: ["top_basic", "bottom_basic"],
      equipped: { hat: null, top: "top_basic", bottom: "bottom_basic", accessory: null, weapon: null }
    },
    ...
  }
}
```
교사 화면(index.html)과 학생 화면(avatar.html) 모두 이 문서를 실시간으로 구독(listen)해서, 누가 수정하든 모든 화면에 바로 반영되도록 한다.
하트(점수)는 기존 `scores` 필드를 그대로 사용 (구매 시 차감).

## 초기 상점 아이템 (예시, 추후 조정 가능)
- **모자**: 야구모자(10), 실크햇(20), 왕관(30), 리본(15)
- **상의**: 민소매(기본,무료), 반팔티(10), 후드집업(20), 멜빵바지 상의(25)
- **하의**: 반바지(기본,무료), 청바지(10), 체크치마(15), 트레이닝바지(15)
- **액세서리**: 안경(10), 목도리(15), 날개(30), 마법지팡이(25)
- **무기** (메이플스토리 무기 종류 참고): 검(20), 도끼(20), 둔기(20), 아대(25), 단검(20), 활(25), 석궁(25), 스태프(30), 완드(30)

## 진행 순서
1. index.html 준비 (기존 파일 복사 + 아바타 이동 버튼 추가)
2. avatar.html 기본 골격 (뒤로가기, 학생 정보 표시, 탭 UI)
3. 아바타 캐릭터 CSS 드로잉 (기본 몸통 + 민소매 + 반바지)
4. 상점 아이템 목록 + 구매 로직
5. 보관함 + 장착/해제 로직
6. 아바타 미리보기 실시간 반영
7. (로컬 버전) 브라우저 테스트 및 다듬기 — 여기까지는 localStorage 기반으로 우선 동작 확인
8. Firebase 프로젝트 생성 안내 (사용자가 직접 구글 계정으로 생성)
9. localStorage 로직 → Firestore 실시간 동기화로 전환
10. 학생 이름 선택(본인 확인) 화면 추가
11. Firebase Hosting 배포 안내 (사용자가 직접 로그인/배포 명령 실행)
12. 여러 기기로 접속 테스트 (교사용 PC + 학생용 폰/태블릿 시뮬레이션)
