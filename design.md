# 디자인 계획 (design.md)

## 전체 톤앤매너
기존 index.html(장등초 자리)과 동일한 느낌을 유지한다.
- 폰트: Noto Sans KR (400/700/900)
- 파스텔 그라데이션 배경 (하늘색 → 보라 → 핑크 → 연두)
- 나무/카드 느낌의 두꺼운 테두리 + 그림자(box-shadow로 입체감)
- 버튼: 주황(기본), 파랑, 빨강, 초록 계열 — 눌렀을 때 눌리는 듯한 그림자 애니메이션
- 상단 툴바: 갈색 배경 + 금색 테두리 (기존과 동일 스타일 재사용)

기존 CSS 변수 그대로 재사용:
```
--bg, --card-bg, --card-border, --desk-bg, --desk-border,
--orange, --orange-dark, --gold, --blue-btn, --navy,
--slime, --slime-dark, --toolbar-bg, --toolbar-border
```

## 아바타 캐릭터 디자인 (CSS 도형)
**메이플스토리 캐릭터를 시각적 레퍼런스로 삼는다.** 메이플스토리는 신규 캐릭터가 기본 속옷(민소매+반바지) 차림으로 시작해서, 옷을 하나씩 장착할 때마다 점점 화려하고 강해 보이는 모습으로 바뀌는 게 특징 — 이 "성장하는 재미"를 그대로 이 프로젝트의 동기부여 요소로 가져온다.

기존 슬라임 캐릭터(.slime-body, .slime-eye 등)와 같은 방식으로 순수 CSS 도형을 쌓아서 사람 캐릭터를 만든다.

구성 요소 (각각 div, absolute 포지셔닝으로 레이어링):
- **머리(head)**: 둥근 원 (살구색), 메이플 캐릭터 특유의 큼직하고 동글동글한 눈 2개 + 입
- **몸통(body)**: 사각형/둥근사각형
- **팔(arms)**: 몸통 양옆 작은 막대
- **다리(legs)**: 몸통 아래 두 막대

기본 착장 (= 메이플스토리 초보자 기본 속옷 컨셉, 무료 시작 아이템):
- **상의 기본 = 민소매**: 몸통 위에 덮는 사각형(어깨 끈만 있는 형태), 팔은 그대로 노출
- **하의 기본 = 반바지**: 다리 상단부만 덮는 짧은 사각형
- 기본 착장 상태는 일부러 밋밋하게 두고, 아이템을 장착할수록 색감·장식이 화려해지도록 설계해 "꾸밀수록 좋아진다"는 느낌을 강조

레이어 순서(z-index, 아래→위):
1. 몸통/팔/다리(맨몸)
2. 하의 (다리 위)
3. 상의 (몸통 위)
4. 무기 (캐릭터 오른손 옆에 비스듬히 배치 — 메이플처럼 종류별로 실루엣이 다름)
5. 액세서리 (안경 등, 얼굴/몸 위)
6. 모자 (머리 맨 위)

아이템 장착 시 해당 부위의 div에 클래스만 바꿔 끼우는 방식 (예: `.top-basic`, `.top-hoodie`, `.hat-crown` 등 각 클래스에 색상/모양 CSS 정의).

**무기(weapon) 종류별 실루엣 컨셉** (CSS 도형으로 단순화해서 표현):
- 검(Sword): 세로로 긴 얇은 직사각형 + 십자 손잡이
- 도끼(Axe): 자루 + 위쪽에 반달 모양 도끼날
- 둔기(Blunt): 자루 + 끝에 둥근 뭉치(별 모양 돌기)
- 아대(Knuckle): 손 부분을 감싸는 짧고 두꺼운 장갑형
- 단검(Dagger): 검보다 짧고 얇은 칼
- 활(Bow): 아치형 곡선 + 시위(활줄)
- 석궁(Crossbow): 십자형 구조 (가로 막대 + 세로 손잡이)
- 스태프(Staff): 긴 막대 + 끝에 큰 보석/장식
- 완드(Wand): 스태프보다 짧은 막대 + 작은 별 장식

각 무기는 이모지(⚔️🪓🏹 등) + CSS 도형을 조합해서 간단히 구현 가능.

## 상점 아이템 카드 UI
```
┌──────────────┐
│   [아이템     │
│    미리보기]  │   ← 작은 CSS 렌더링 or 이모지
│  야구모자     │
│  ❤️ 10       │
│  [구매하기]   │  ← 이미 보유시 "보유중" 배지, 장착중이면 강조 테두리
└──────────────┘
```
- 하트 부족하면 버튼 비활성화(회색) + 클릭 시 토스트 "하트가 부족해요!"
- 보관함 탭에서는 가격 대신 [장착]/[해제] 버튼 표시

## 인터랙션 디테일
- 아이템 구매/장착 시 아바타 미리보기에 살짝 반짝이는 애니메이션 (기존 slime bounce처럼)
- 하트 차감 시 상단 하트 숫자에 flash 애니메이션 (기존 score-num.flash-down 재사용)
- 토스트 메시지 스타일은 기존 .toast 그대로 재사용
